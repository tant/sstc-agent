# Using Local Telegram Bot Library

## Overview

This project uses a local copy of the node-telegram-bot-api library instead of the npm package to avoid dependency conflicts and allow for customization.

## Directory Structure

```
src/
└── lib/
    └── node-telegram-bot-api/     # Local copy of node-telegram-bot-api library
        ├── src/                   # Source code directory
        ├── lib/                   # Transpiled code directory (generated by build)
        ├── index.js
        └── package.json
src/
└── mastra/
    └── channels/
        └── telegram/              # Telegram channel implementation using local library
```

## Removed Files

The following files and directories have been removed as they are not needed for production:
- Configuration and metadata files
- Documentation files
- Build configuration files

## Dependency Management

"dependencies": {

These dependencies have been added to the main project's `package.json` to ensure they are available.

## Building the Local Library

To build the local library, run the following commands:

```bash
cd src/lib/node-telegram-bot-api
pnpm install
pnpm run build
```

This will generate the transpiled code in the `lib/` directory.

## Updating the Local Library

To update the local node-telegram-bot-api library:

1. Pull the latest changes from the node-telegram-bot-api repository
2. Copy the updated files to `src/lib/node-telegram-bot-api/`
3. Update dependencies in the main `package.json` if needed
4. Run `pnpm install` to install any new dependencies
5. Run `pnpm run build` in the library directory to build the transpiled code

## Customization

Since we're using a local copy, you can customize the node-telegram-bot-api library directly:

1. Add logging for debugging purposes
2. Modify API behavior to better suit your needs
3. Fix bugs without waiting for upstream updates

## Known Issues

The local library and its dependencies have several security vulnerabilities, but these are inherited from the original library and its outdated dependencies. These vulnerabilities are:

- Critical vulnerabilities in growl, underscore, babel-traverse, minimist, and form-data
- High vulnerabilities in taffydb, node-static, debug, marked, shelljs, json5, braces, and diff
- Moderate vulnerabilities in node-static, marked, shelljs, minimist, request, tough-cookie, ajv, and micromatch
- Low vulnerabilities in braces and debug

These vulnerabilities are primarily due to outdated dependencies in the library. In a production environment, you should carefully evaluate the risk of these vulnerabilities.

## Benefits

1. **Dependency Conflict Resolution**: Avoids conflicts with other libraries that may require different versions of the same dependencies
2. **Performance**: No need to fetch from npm registry
3. **Customization**: Full control over the library's behavior
4. **Debugging**: Easier to debug and trace issues within the library
